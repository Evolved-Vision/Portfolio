Projects

Network Vulnerability Assessment Using Wireshark & Nmap

Explored network vulnerabilities by scanning hosts using Nmap and analyzing traffic with Wireshark, identifying risks such as open ports, brute-force attacks, and unencrypted transmissions.

Tools and Techniques:

NMAP: Used for network discovery, topology mapping, and vulnerability identification.

Wireshark: Captured and analyszed traffic, uncovering unauthorized access attempts and potential DoS attacks.

Security Hardening Recommendations: Applied best practices from NIST, OWASP, and industry standards.

Key Findings:

FTP Anonymous Login Vulnerability – Allowed unauthorized access to sensitive files.
Port 80 Unencrypted Traffic – Exposed credentials to interception attacks.
Brute Force & DoS Indicators – Repeated failed login attempts and repetitive HTTP requests suggested ongoing attacks.

Mitigation Strategies:

Enforced HTTPS (Port 443) and blocked Port 80.
Implemented login throttling and MFA to prevent brute-force attacks.
Applied rate limiting and firewall rules to mitigate DoS threats.

Evidence & Screenshots:

I completed the nmap 10.168.27.0/24 command to complete an initial network discovery.

Afterward, to identify the topology I completed nmap –traceroute 10.168.27.0/24 to get an idea of the path of traffic on the network.

After reviewing the output all host that are up on the network only have 1 hop to the gateway which indicates a star topology.

After reviewing the output for Nmap it appears that there are two host that has several ports open to unknown services.
These could represent security risk within the network. Further investigation would be needed to gather additional information
regarding the legitimacy of the services running on the open ports. The screenshot shows that there is an ftp anonymous log in,
which could allow unauthorized access to sensitive files on the server. There is a risk of data theft, unauthorized modification,
or exploitation of the server. Additionally, port 80 is open, which is an unencrypted port. This can mean that traffic transmitted over
this port such as log in credentials and or other sensitive data, could be intercepted by attacks such as man in the middle attacks or
modification of transmitted data.

Furthermore, the http responses for the status code “200 ok” all have the same byte length of 1950 bytes.
This appears to be a request of the same resource over and over which could also indicate a DOS attack.
This constant request of the same resource could also imply that a bot is being used to make these repetitive
request in order to overload the system.
